MeetingScribe — Privacy Policy

Last updated: April 10, 2026

What MeetingScribe Does

MeetingScribe is a Chrome browser extension that helps you take notes on calendar meetings and save them to your own cloud storage (Google Drive, OneDrive, or Dropbox). It connects to your calendar (Google Calendar or Microsoft Outlook) to show upcoming meetings and pre-fill note templates.

Data We Collect

We do not collect any personal data. MeetingScribe has no backend server, no analytics, and no tracking. Specifically:

  • No user data is transmitted to any server controlled by MeetingScribe or its developers.
  • No usage analytics, crash reports, or telemetry are collected.
  • No cookies are set by MeetingScribe.
  • No advertising or marketing data is collected.

Data We Access

MeetingScribe accesses the following data solely to provide its core functionality:

  • Calendar events — Read from your Google Calendar (via Google Calendar API) or Microsoft Outlook (via Microsoft Graph API) to display upcoming meetings and pre-fill note templates. Events are cached locally in chrome.storage.local for up to 5 minutes.
  • OAuth tokens — Stored in chrome.storage.local on your machine. Used to authenticate API calls to Google, Microsoft, and Dropbox on your behalf. Tokens are never transmitted to any third party.
  • Meeting notes — Content you type in the editor. Auto-saved as drafts in chrome.storage.local. When you click Save, notes are uploaded directly to your own cloud storage account — never to any MeetingScribe-controlled storage.

Data Storage

All data is stored locally on your device in chrome.storage.local. This includes:

  • Connected account information (email, display name, OAuth tokens)
  • Extension settings (routing rules, preferences)
  • Draft notes (auto-saved editor content)
  • Cached calendar events

You can clear all stored data at any time by removing the extension from Chrome.

Third-Party Services

MeetingScribe communicates directly with the following services using your own authenticated accounts:

  • Google — Calendar API (read events), Drive API (upload notes), OAuth 2.0 (authentication)
  • Microsoft — Graph API (read events, upload to OneDrive), OAuth 2.0 (authentication)
  • Dropbox — HTTP API v2 (upload notes), OAuth 2.0 (authentication)

Each of these services has its own privacy policy. MeetingScribe does not control how these services handle your data beyond what their APIs provide.
